What I do?

Always aiming to be a full stack developer on a project.
Depending on the team requirement and how interesting and challenging the task is, I will work on backend or front-end.

  • Front-End

    Front-end or User Interface is one of the most important aspect of any application. I try to keep myself udpated with front-end skills and knowledge and give it due importance on any project.

  • Backend

    A good front end require a strong backend with high performance. Writing clean and flexible code and getting the business logic right is critical. I started my backend journey with JAVA and moved on to Ruby on Rails. Will be working on Python next.

  • UX

    A good user interface is the one which does not require a tutorial to learn. Over the few years i've worked, I've paid close attention to UX heuristics to keep the design clean and user friendly. I won't claim to be an expert, but I can certainly build usable interfaces.

  • Devops

    I find devops an important skill for everyone who want's to build end to end systems. I've not dived into devops much yet. I've taken my first step by attending meetup's and sessions about Devops tools like Chef and Ansible. It's an exciting field and I'm looking forward to learn more.

My Skills

I enjoy programming. My goal is to be an end to end, full stack developer capable of working at any level.

HTML85%

Whenever writing markup comes into play, I give a lot of importance to semantics. I try to separate the Content from presentation and keep the markup simple. I still need to learn the advanced features that HTML-5 brings into the fold.

CSS85%

CSS is what brings the page alive in my opinion. It's what makes the page look beautiful. Not many people posses good CSS skills. Also, keeping the CSS code to minimum is very important for performance in the long run as i've learnt from my recent experiences.

JAVASCRIPT75%

Javascript has evolved a lot. I use it as a first class language. As Douglas Crockford says it, Javascript is The World's Most Misunderstood Programming Language. It's one language for everything (client side and server side). I'm still learning a lot about it.

RUBY ON RAILS60%

Ruby on Rails is one of the best platforms to build a website in no time. It's dynamic nature can be tricky but it's very powerful. I've enjoyed my time working on a RoR project for a year. It's still loved by

PREPROCESSORS90%

Writing vanila CSS or HTML can slow you down sometimes. That's where preprocessors like SASS, LESS, HAML, SLIM, JADE etc. come to rescue. It helps a lot to increase productivity. I've used these extensively.

FRONTEND PERFORMANCE80%

One of the most important skills for any frontend developer is the ability to keep the performance of client side code at it's best. I've had some amazing experiences in recent times working explicitely on client side performance.

UX40%

User experience is an art in some sense. It's vital for any application to have a simple and efficient user interface. I try my hand at it from time to time. My user journeys are receiving some positive responses.

ANDROID30%

Mobile apps are the future. With the boom of internet of things, I am very excited to give android development a shot. I've mostly done basics and I'm still in self learning mode. It's interesting.

PUBLIC SPEAKING40%

My Journey so far..

Under Graduation: May 2012

I completed my undergraduation from Pune Institute of Computer Science in Information Technology. I achieved First Class with Distinction throughout. My favourite subjects were Computer Networks, Algorithms, Data Structures and Operating Systems.

Joined ThoughtWorks: July 2012

After undergrad, I joined ThoughtWorks (TW) as an application developer. I was one of the chosen 4 out of over 300 students to get selected. ThoughtWorks was a perfect start to my career because at ThoughtWorks, I get to learn something new every single day. It's a place where you can never be the smartest person in the room.

First Frontend Experience: April 2013

Soon after joining ThoughtWorks I learnt frontend skills and got staffed as the only frontend developer for a client project in TW-Delhi. It's where I lead the frontend development on my own. It was a small team and I got to learn a hell lot from that experience.

Added Ruby on Rails to my skills: Oct 2013

My burning desire to be a full stack developer took a great turn when I started working on RoR for a big project. Originally staffed as a frontend developer, it was difficult to get the right support and work on both frontend and backend. I learnt basic RoR on my own and then got support from my team and a few months later was able to call my self a RoR developer.

Rails Girls coach: December 2014

Owing to my learning on the RoR project, it was time to give back to the community. I signed up as a Rails Girls coach. The event took place in ThoughtWorks, Pune. In about 3 hours me and my team of trainees deployed a working feedback application to Heroku. It got even better for me when I received extremely positive feedback from my trainee.

Building "Right To Left": March 2015

A multi-national, multi-locale marketing website with thousands of pages. We had to support the entire website in Middle East countries and support Right To Left languages like Arabic. This was probably the first time ever in ThoughtWorks that we got this unique requirement. Learnt a lot about I18 (internationalization). I also worked on Performance of the website which was most challenging.
